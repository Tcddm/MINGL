# 裁剪系统
## 核心职责与设计哲学
MINGL的裁剪系统是“按组件刷新”机制的基石。它解决了一个核心问题：当一个控件需要重绘时，如何保证它只绘制自己该画的那部分像素，而绝不越界？

在传统 GUI 框架中，裁剪通常通过全局脏矩形合并来实现——收集屏幕上所有变化区域，合并成大矩形，然后在这个矩形内重绘所有涉及的控件。这种方式需要维护全局脏矩形列表，且合并可能引入过度绘制。

MINGL选择了一条更精细，更简洁的路：裁剪传递。

**裁剪上下文**（mgl_draw_ctx_t）是整个系统的核心数据结构。它内部持有一个裁剪矩形clip，表示当前控件允许绘制的屏幕区域。这个矩形随着控件树的递归遍历，从根节点向叶子节点层层传递，并在每一层被控件的自身边界与父级裁剪区域求交，逐步缩小。

当控件的draw虚函数被调用时，它已经拥有了一个精确到像素的裁剪上下文。控件不需要关心自己的哪一部分被遮挡，哪一部分在屏幕外，它只需要照常绘制，所有mgl_ctx_xxx函数内部会自动用ctx->clip与目标矩形求交，过滤掉越界的像素。

这种设计的核心优势在于：

- **精准性**：每个控件只绘制自己被允许绘制的像素。

- **简洁性**：控件开发者不需要在绘制代码中处理任何裁剪逻辑，框架自动完成。

- **局部性**：每个控件的裁剪区域由它自己的边界和父级传入的裁剪区域共同决定，不需要任何全局信息。这意味着裁剪系统是完全自洽的——你可以单独测试任何一个控件的绘制，只要给它一个裁剪上下文，它就能正确工作。
